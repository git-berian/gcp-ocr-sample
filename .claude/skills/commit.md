# コミット規則

## テンプレート

`.github/commit_template.md` の形式に従う：

```
<type>(<scope>): <summary>

<本文（任意）>

関連: #<issue番号>
```

## ルール

- Conventional Commits の形式を使用する
- 変更内容から type と scope を適切に推測する
- 日本語で記述する
- summary は50文字以内に収める
- 本文は任意。必要に応じて変更の補足を自由記述で書く
- 詳細な変更内容・変更理由・影響範囲は PR に記載する（Squash merge 運用のため）
- 1コミット1作業に分ける（複数作業の内容を1コミットに含めない）
- コミットメッセージは実行前にユーザーに提示して確認を取る

## type 一覧

| type       | 用途               |
| ---------- | ------------------ |
| `feat`     | 新機能             |
| `fix`      | バグ修正           |
| `refactor` | リファクタリング   |
| `docs`     | ドキュメント       |
| `chore`    | 設定・依存関係など |
| `test`     | テストの追加・修正 |

## 注意

- `tasks/` ディレクトリは `.gitignore` に含まれておりローカル専用。`git add` に含めない
- コミット前に `.gitignore` を確認し、管理対象外のファイルをステージしない
- commitlint により、PR 作成時に CI でコミットメッセージが自動検証される
